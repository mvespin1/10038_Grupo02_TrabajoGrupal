Introducción a XML

¿Qué es XML?


El lenguaje de marcado extensible (XML) permite definir y almacenar datos de forma compartible. XML admite el intercambio de información entre sistemas de computación, como sitios web, bases de datos y aplicaciones de terceros. Las reglas predefinidas facilitan la transmisión de datos como archivos XML a través de cualquier red, ya que el destinatario puede usar esas reglas para leer los datos de forma precisa y eficiente.

  • XML significa lenguaje de marcado extensible
  • XML es un lenguaje de marcado muy parecido a HTML
  • XML fue diseñado para almacenar y transportar datos
  • XML fue diseñado para ser autodescriptivo

¿Por qué es importante el XML?


El lenguaje de marcado extensible (XML) es un lenguaje de marcado que proporciona reglas para definir cualquier dato. A diferencia de otros lenguajes de programación, XML no puede realizar operaciones de computación por sí mismo. En cambio, se puede implementar cualquier software o lenguaje de programación para la administración estructurada de datos.
Por ejemplo, imagine un documento de texto con comentarios. Los comentarios pueden ofrecer sugerencias como las siguientes:


  • Ponga el título en negrita
  • Esta oración es un encabezado
  • Esta palabra es el autor

Estos comentarios mejoran la usabilidad del documento sin repercutir en su contenido. Del mismo modo, XML utiliza símbolos de marcado para proporcionar más información sobre los datos. Otros programas, como los navegadores y las aplicaciones de procesamiento de datos, utilizan esta información para procesar datos estructurados de manera más eficiente.

Como Utilizar XML

XML se utiliza en muchos aspectos del desarrollo web.
XML se usa a menudo para separar los datos de la presentación.
XML separa los datos de la presentación
XML no contiene ninguna información sobre cómo se mostrará.
Los mismos datos XML se pueden utilizar en muchos escenarios de presentación diferentes.
Debido a esto, con XML, existe una separación total entre datos y presentación.
XML suele ser un complemento de HTML
En muchas aplicaciones HTML, XML se usa para almacenar o transportar datos, mientras que HTML se usa para formatear y mostrar los mismos datos.
XML separa los datos de HTML
Al mostrar datos en HTML, no debería tener que editar el archivo HTML cuando cambien los datos.
Con XML, los datos se pueden almacenar en archivos XML separados.
Con unas pocas líneas de código JavaScript, puede leer un archivo XML y actualizar el contenido de datos de cualquier página HTML.

Introducción a JSON
¿Qué es JSON?

JSON (JavaScript Object Notation) o "Notación de Objetos de JavaScript" por su traducción al español consiste en un formato de intercambio de datos ligero y legible por humanos, que es utilizado para almacenar una colección de pares nombre-valor o una lista ordenada de valores.


Principales Características de JSON

  • JSON es completamente independiente del lenguaje, es decir que puede ser utilizado por una amplia variedad de lenguajes como JavaScript, Python, Java, C#, entre otros.
  • JSON es utilizado para enviar datos entre computadoras
  • JSON admite una variedad de tipos de datos, incluyendo cadenas de texto, números, booleanos, null, objetos y arreglos.
  • JSON puede ser interpretado y generado por diferentes plataformas y sistemas operativos.

¿Por qué es importante JSON?

Entre las principales ventajas del uso de JSON, se tiene:

  • Al ser un formato de texto plano, los datos JSON se pueden enviar y recibir fácilmente entre diferentes computadoras y sistemas.
  • El formato JSON es fácil de leer y entender tanto para los desarrolladores como para los sistemas.
  • Debido a que el formato JSON se asemeja a la sintaxis para crear objetos en JavaScript, los datos JSON se pueden convertir fácilmente en objetos JavaScript.
  • JSON es ampliamente utilizado en el desarrollo web y en servicios web API. Debido a su simplicidad y facilidad de uso, se ha convertido en un estándar de facto en la transmisión y almacenamiento de datos estructurados en aplicaciones web.
  • JavaScript proporciona funciones integradas para convertir datos JSON en objetos JavaScript "JSON.parse()" y para convertir objetos JavaScript en cadenas JSON "JSON.stringify()".
Sintaxis de JSON

JSON utiliza una sintaxis similar a la notación de objetos en JavaScript. Esta cuenta con los siguientes aspectos:

  • Está compuesto por pares "clave-valor" y listas de valores.
  • Los datos se encuentran separados por comas.
  • Los corchetes se caracterizan por contener a matrices.
  • Las llaves contienen objetos.

SINTAXIS:

Los datos JSON se escriben como pares de clave-valor, los cuales estarán dentro de comillas dobles (" "), y se encontrarán separados por dos puntos (:) como se lo indica a continuación:

{"nombre":"Jason"}

La sintaxis de JSON es muy parecida a la sintaxis de los objetos de JavaScript, solo que en el segundo caso, no son necesarias utilizar las comillas para declarar a las claves.

{nombre:"Jason"}

Valores en JSON

Los valores que pueden ser asignadas a las claves en Json, pueden ser las siguientes:

  • Objetos.
  • Valores Booleanos.
  • Valores String.
  • Valores Numéricos.
  • Arreglos.
  • Valores nulos o "null".

Objetos en JSON.

{
"persona":{"nombre":"Jason", "edad":22, "pais":"Ecuador"}
}

Valores Booleanos en JSON.

{"sale":true}

Valores String en JSON.

{"nombre":"Jason"}

Valores Numéricos en JSON.

{"edad":22}

Arreglos en JSON.

{
"estudiantes":["Jason", "Mario", "Peter"]
}

Valores nulos o "null" en JSON.

{"nombre":null}
JSON.parse()

Por lo general se utiliza JSON para intercambiar datos con un servidor web, los cuales al ser recibidos siempre están en forma de una cadena de texto.

Es por esto que se puede utilizar la función JSON.parse(), la cual se caracteriza por analizar los datos y los convierten en un objeto JavaScript.


Se puede convertir un texto con formato JSON en un objeto JavaScript con la función JSON.parse(), de la siguiente manera:

'{"nombre":"Jason", "edad":22, "pais":"Ecuador"}'

Utilizando la función JSON.parse() para convertirlo en un objeto JavaScript, se tiene:

const persona = JSON.parse('{"nombre":"Jason", "edad":22, "pais":"Ecuador"}');

Ejemplo (Convertir texto a objeto y extraer los valores de las claves):

const textoJson1 = '{"nombre":"Jason", "edad":22, "pais":"Ecuador"}';
const persona1 = JSON.parse(textoJson1);
document.getElementById("demoJson1").innerHTML = persona1.nombre + " tiene la edad de: " + persona1.edad + " años.";

JSON.stringify()

Por lo general se utiliza JSON para intercambiar datos con un servidor web, los cuales al ser recibidos siempre están en forma de una cadena de texto.

Es por esto que se puede utilizar la función JSON.stringify(), la cual se caracteriza por convertir un objeto JavaScript a un texto con formato JSON.


Se puede convertir un objeto JavaScript a un texto con formato JSON con la función JSON.stringify(), de la siguiente manera:

const persona = JSON.stringify('{"nombre":"Jason", "edad":22, "pais":"Ecuador"}');

Utilizando la función JSON.stringify() para convertirlo en un objeto JavaScript, se tiene:

const objetoJSON = JSON.stringify(persona);

Ejemplo (Convertir objeto a texto y escribir el texto):

const persona2 = {nombre:"Jason", edad:22, pais:"Ecuador"};
const textoJson2 = JSON.stringify(persona2);
document.getElementById("demoJson2").innerHTML = textoJson2;

Introducción a AJAX
¿Qué es AJAX?

Es un conjunto de técnicas de desarrollo web que permiten que las aplicaciones web funcionen de forma asíncrona, procesando cualquier solicitud al servidor en segundo plano.

  • AJAX = J avaScript asincrónico y X ML.
  • AJAX no es un lenguaje de programación.
AJAX solo usa una combinación de: Un objeto integrado en el navegador XMLHttpRequest (para solicitar datos de un servidor web) JavaScript y HTML DOM (para mostrar o usar los datos).

Características clave de AJAX
  • Asincronía:

    • Permite realizar solicitudes al servidor de forma asíncrona, lo que significa que la página web puede seguir siendo interactiva mientras se espera la respuesta del servidor.

  • Actualizaciones parciales:

    • AJAX permite actualizar solo partes específicas de una página web, en lugar de recargarla por completo. Esto mejora la experiencia del usuario y reduce la carga en el servidor.

  • Intercambio de datos:

    • AJAX facilita la transferencia de datos entre el navegador y el servidor en diferentes formatos, como XML, JSON o texto plano.

  • Manipulación del DOM:

    • Permite manipular el Document Object Model (DOM) de una página web para actualizar contenido dinámicamente sin necesidad de recargarla.

    Como funciona AJAX

    AJAX permite que las páginas web se actualicen de forma asíncrona mediante el intercambio de datos con un servidor web en segundo plano.
    Esto significa que es posible actualizar partes de una página web sin recargar toda la página.



    1. Se produce un evento en una página web (se carga la página, se hace clic en un botón)
    2. JavaScript crea un objeto XMLHttpRequest
    3. El objeto XMLHttpRequest envía una solicitud a un servidor web
    4. El servidor procesa la solicitud
    5. El servidor envía una respuesta a la página web.
    6. La respuesta es leída por JavaScript
    7. JavaScript realiza la acción adecuada (como la actualización de la página)
